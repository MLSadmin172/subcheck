# SubCheck

MVP-сервис, где пользователь за единоразовую оплату получает доступ к списку своих подписок.

## Что реализовано

- Лендинг с описанием единоразовой оплаты.
- Флоу "ввод email -> оплата -> доступ в кабинет".
- Кабинет со списком подписок, категориями и суммой расходов.
- Импорт реальных подписок из почты через IMAP (Gmail/Яндекс/Mail и др.).
- Добавление подписки вручную.

## Быстрый старт

1. Установить Python 3.10+.
2. В папке проекта выполнить:

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
python app.py
```

3. Открыть `http://127.0.0.1:5000`.

## Подключение почты

### Gmail (рекомендуется)

1. Нажми "Подключить Gmail".
2. Разреши доступ в Google.
3. Нажми "Синхронизировать Gmail сейчас".

### Mail.ru

1. Создай пароль приложения в Mail.ru.
2. В кабинете SubCheck введи логин Mail.ru и app password.
3. Нажми "Подключить Mail.ru" и затем "Синхронизировать Mail.ru сейчас".

Сервис читает письма и ищет признаки подписок по отправителю/теме/тексту.

## Подписки по банковской карте

Можно проверить подписки по конкретной карте:

1. Укажи последние 4 цифры карты.
2. Загрузи выписку CSV или PDF.
3. Сервис найдет регулярные списания и покажет подписки с пометкой `*1234`.

### Зафиксированный профиль парсинга PDF

Для текущего формата банковской PDF-выписки в коде зафиксирован рабочий профиль:

- операция начинается со строки `dd.mm.yyyy hh:mm`;
- сумма операции берется как первый расходный денежный токен в блоке;
- известные сервисы (например `Wink`, `KinoPoisk`) ищутся и по блоку, и по локальному fallback-контексту.

## Самый простой UX (Gmail OAuth)

Чтобы пользователь не вводил IMAP/app password:

1. В кабинете нажать "Подключить Gmail".
2. Разрешить доступ в Google.
3. Сервис автоматически подтянет подписки по письмам.

Для этого нужно один раз настроить переменные окружения:

- `GOOGLE_CLIENT_ID`
- `GOOGLE_CLIENT_SECRET`

Или (проще) прямо в кабинете ввести `Client ID` и `Client Secret` в форме "Google OAuth настройки".

## Важно

- Распознавание подписок эвристическое (по ключевым словам и доменам), поэтому возможны пропуски.
- Для продакшена нужно: реальный платежный провайдер, шифрование секретов и защищенное хранение токенов/паролей.

## Реальная оплата (Stripe)

Для включения Stripe Checkout:

- `STRIPE_SECRET_KEY` — секретный ключ Stripe
- `STRIPE_PRICE_ID` — id цены (one-time) в Stripe
- `STRIPE_WEBHOOK_SECRET` — секрет webhook endpoint
- `BASE_URL` — публичный URL сервиса (`https://...`)

Webhook endpoint:

- `POST /payments/stripe/webhook`

Если Stripe переменные не заданы, сервис использует локальный fallback-режим оплаты (для разработки).

## Публикация в открытый доступ (Render)

В проект добавлен `render.yaml` для деплоя.

Базовый запуск в продакшене:

- `gunicorn -w 2 -b 0.0.0.0:$PORT app:app`
